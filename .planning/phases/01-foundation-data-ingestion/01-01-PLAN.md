<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User -> REST API | File uploads cross trust boundary -- untrusted input |
| API -> Celery | Task dispatch is internal but passes file paths |
| Application -> Docker services | Qdrant/Redis/PostgreSQL on localhost -- no network exposure |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | File upload endpoints | mitigate | File type validation (magic bytes), size limit (100MB via config), UUID filenames (never user-supplied paths) |
| T-01-02 | Denial of Service | File upload endpoints | mitigate | MAX_FILE_SIZE_MB=100 config limit, reject oversized files early |
| T-01-03 | Information Disclosure | docker-compose.yml | accept | Default credentials (vbrain:vbrain) for local dev only. Production must override via env vars |
| T-01-04 | Information Disclosure | .env file | mitigate | .gitignore includes .env -- secrets never committed. Only .env.example in repo |
| T-01-05 | Tampering | docker-compose exposed ports | accept | Local dev only -- ports bound to localhost. Production deployment uses Docker networks |
</threat_model>
