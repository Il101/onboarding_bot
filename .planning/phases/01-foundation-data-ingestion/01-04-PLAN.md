ЗАМЕЧАНИЕ: Пресидио OperatorConfig("replace", {"new_value": "<PERSON_>"}) заменит каждое вхождение на "<PERSON_>". Для консистентной нумерации (<PERSON_1>, <PERSON_2>) нужно использовать TokenMapper. Реализовать custom Operator или post-processing шаг, который заменяет "<PERSON_>" на "<PERSON_N>" через TokenMapper.

Подход: Использовать custom Anonymizer operator или post-process результат:
1. Запустить Presidio anonymizer с placeholder-оператором (заменяет на уникальный маркер)
2. Использовать TokenMapper для замены каждого уникального маркера на токен с номером

Конкретная реализация: Использовать OperatorConfig("replace", {"new_value": ""}) для каждого типа, затем во втором проходе через TokenMapper пронумеровать каждое вхождение. Либо использовать custom OperatorConfig("custom", {"class": CustomOperator}) -- см. Presidio docs.

Простейший рабочий подход:
1. Получить analyzer_results (список RecognizerResult с start, end, entity_type, score)
2. Для каждого результата: извлечь original_value = text[start:end], получить токен из TokenMapper
3. Заменить в тексте original_value на токен
